# One-Page Demo Walkthrough

## Demo URL

https://securethecloud-mcp-governance-lab.fly.dev

## Opening

This demo shows how AI agents can be governed before they invoke MCP-style enterprise tools.

The central idea is simple:

AI can assist and recommend, but tool execution must pass through identity, policy, approval, firewall, and evidence controls first.

## Step 1 — Public Demo Boundary

Point out that the lab is simulated and does not connect to real enterprise systems or customer records.

This makes the demo safe, honest, and client-ready.

## Step 2 — Shared Trust Fabric

Explain the five control areas:

- Governance & Policy
- Evidence & Audit
- Risk Intelligence
- Identity / Context
- MCP Tool Firewall

## Step 3 — Executive Risk & Control Center

Show leadership-level visibility into:

- Risk distribution
- Decision distribution
- Tool-call firewall summary
- Sensitive attempts
- Policy violations
- Human approval pressure
- Readiness posture

## Step 4 — MCP Access Request Portal

Use the request portal to show how a governed MCP-style tool request includes:

- User identity
- Role
- Department
- Business unit
- Clearance level
- Agent identity
- Service identity
- Environment context
- Requested MCP tool
- Requested resource
- Data classification
- Approval status
- Business purpose

## Step 5 — MCP Tool-Call Firewall

Show that every tool receives a decision:

- Allowed
- Denied
- Redacted
- Requires approval
- Escalated

The firewall explains why each tool is allowed or blocked.

## Step 6 — Human Approval Workspace

Show how sensitive requests can pause for human review before execution is allowed.

## Step 7 — Evidence Replay

Show the evidence timeline:

- Request intake
- Identity context
- Data classification
- Risk tiering
- Policy decision
- Human approval
- Final outcome

## Step 8 — Mock MCP Execution Adapter

Explain that the adapter simulates final execution only after the governance checks pass.

If checks fail, execution is blocked, paused, redacted, or escalated.

## Closing

This demo is valuable because AI agents increasingly need to use tools. Enterprises need a governed checkpoint before those tools can touch sensitive data, workflows, or business systems.
