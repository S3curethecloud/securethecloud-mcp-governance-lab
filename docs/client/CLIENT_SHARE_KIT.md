# SecureTheCloud MCP Governance Lab — Client Share Kit

## Public Demo

https://securethecloud-mcp-governance-lab.fly.dev

## One-Line Summary

SecureTheCloud MCP Governance Lab is a simulated AI governance control plane showing how MCP-style tool access can be governed before AI agents retrieve sensitive information, invoke enterprise tools, or perform business actions.

## Short Client-Safe Description

This demo shows a production-shaped governance workflow for AI tool access. It models identity context, data classification, risk tiering, policy decisions, MCP-style tool-call firewalling, human approval, mock execution, evidence replay, and executive governance visibility.

The lab is intentionally simulated. It does not connect to real MCP servers, real customer records, regulated data, production authorization systems, or enterprise infrastructure.

## Core Demo Claim

AI can assist, recommend, and explain.

AI cannot invoke MCP-style tools until identity, policy, approval, firewall, and evidence checks pass.

## What the Demo Shows

- Public demo boundary and correct claim
- Shared trust fabric
- Executive risk and control dashboard
- MCP access request portal
- MCP tool-call firewall
- Human approval workspace
- Evidence replay timeline
- Protected demo reset
- Mock MCP execution adapter

## Suggested Demo Flow

1. Open the public demo URL.
2. Explain the core principle: AI cannot bypass governance.
3. Show the public demo boundary to establish honest scope.
4. Walk through the executive risk dashboard.
5. Submit or inspect a sensitive MCP-style tool request.
6. Show the tool-call firewall outcome.
7. Show approval-required routing.
8. Show evidence replay.
9. Run or explain the mock execution adapter.
10. Close with the enterprise value: governed AI tool access before execution.

## Correct Scope

This is a simulated governance lab and public product demo. It is designed to demonstrate architecture, workflow, and control-plane thinking.

## Not Claimed

This demo is not a production enforcement system, SOC 2 certification, real MCP server, real customer-data processor, or real enterprise authorization layer.
